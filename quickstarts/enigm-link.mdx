Enigm Link is delivered as an official sealed USB product governed through Enigm Command. This quickstart explains the user-facing flow for opening the protected environment, using the secure browser, updating the USB, and starting the isolated Boot OS mode where supported.

Enigm Link is a closed product. Users do not install Enigm Link, rebuild the USB, provision boot media, or replace internal files. The user workflow is limited to opening the official USB, selecting the supported mode, and following the product prompts.

## What You Need

Before starting, confirm that you have:

- The official Enigm Link USB.
- A supported computer with a direct USB port.
- Network connectivity.
- An Enigm Link entitlement managed through Enigm Command.

Do not copy the app to another USB, reformat the device, rename product folders, or modify internal files. Official USB validation is part of the Enigm Link trust model.

## Open The Secure Browser

Open Enigm Link from the official USB. The desktop app prepares the protected environment and presents two primary modes:

- **Secure browser**: opens Enigm Command in an isolated temporary browser window.
- **Secure boot**: guides compatible computers into the Enigm Link Boot OS environment.

Use **Secure browser** when you want to access Enigm Command from the protected desktop environment without rebooting the computer.

## Confirm Protected State

The secure browser flow should show that the temporary session is prepared and that the environment is protected.

The protected state indicates that Enigm Link has prepared the controlled environment and validated the required product state. A normal network connection alone is not enough; protected state depends on Enigm Link environment preparation and product validation.

After protection is ready, select **Open Enigm Command**. Enigm Link opens `https://app.enigm.io` in an isolated window with a temporary browser profile controlled by Enigm Link.

## Update Enigm Link

If an update is available, Enigm Link presents an **Update Enigm Link** action inside the product interface.

<div className="enigm-doc-image-frame">
  <img className="enigm-doc-image" src="/assets/images/link/enigm-link-update-available.png" alt="Enigm Link update available action" />
</div>

Use the in-product update action only. Do not replace files manually.

<Warning>
  Enigm Link updates can take approximately five minutes. Keep the USB connected, wait for the app to close and reopen automatically, and do not interrupt the update while the USB is being updated.
</Warning>

During update:

- Keep the USB connected.
- Keep the app open.
- Wait until the update state completes.
- Do not rename folders or move files while the update is running.

The update flow is designed to preserve official USB identity and apply authorized product updates through the Enigm Link lifecycle model.

## Finish Safely

When the session is complete:

1. Sign out of Enigm Command.
2. Close the secure browser or shut down from the protected Boot OS environment.
3. Wait until the computer is fully powered off before removing the USB.

Removing the USB during an active update or boot session can interrupt the protected environment.

## Troubleshooting

Common user-facing cases:

- **The environment is not protected**: wait for Enigm Link to finish preparing the environment, then retry from the official USB.
- **The update button appears**: run the update from inside Enigm Link and keep the USB connected until completion.
- **The USB appears twice in the boot menu**: select the `UEFI` entry.
- **The boot menu does not appear**: restart and try the boot-menu keys shown by the product flow.
- **Apple Silicon does not show USB boot**: use the normal macOS Enigm Link app. Boot OS USB is not supported on Apple Silicon in the current build.

For more detail, see [Link Troubleshooting](/link/troubleshooting).

## Security Boundaries

Enigm Link provides a controlled Enigm access environment, not a guarantee that every host computer, website, or user action is safe.

Enigm Link does not replace:

- Enigm App end-to-end encryption.
- Enigm Command account controls.
- User verification decisions.
- Host-device security awareness.
- Safe handling of downloaded files or credentials.
