This quickstart explains the user-facing Enigm OS first-run setup flow. The wizard establishes the initial device baseline before normal use, including language selection, connectivity, time configuration, optional biometric enrollment, appearance, and transition into the Enigm experience.

Enigm OS setup is designed to be direct and security-focused. It avoids general-purpose account ecosystems and keeps onboarding centered on the controls required to start the device safely.

## What You Need

Before starting, confirm that you have:

- A supported Fairphone 6 Enigm OS device with sufficient battery.
- Network access through Wi-Fi, mobile connectivity, or the deployment network available to the device.
- The credentials or enrollment information required by your Enigm deployment.
- Physical control of the device during setup.

Enigm OS is currently supported only on Fairphone 6. This quickstart should not be used as production guidance for unsupported device models.

During onboarding, complete any device authentication, PIN, or enrollment screens presented by the device before continuing into normal use.

## Security Notes

The setup wizard establishes the initial device baseline, but it does not replace the broader Enigm security model.

- Enigm App message confidentiality remains based on protected key material and end-to-end encryption.
- Device Trust depends on Enigm OS security services, Trust Security Center, policy state, and controlled update posture.
- Biometric enrollment is a local device unlock method, not an Enigm account recovery mechanism.
- Administrative management does not provide access to message plaintext, call content, media, attachments, or private key material.

See [OS Device Experience and Management](/os/device-experience-management) and [Trust Security Center](/os/trust-security-center) for the architecture model behind setup, device posture, and trust visibility.
