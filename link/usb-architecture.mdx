The official Enigm Link USB is the sealed physical distribution and identity boundary for Enigm Link. It is delivered by Enigm as a prepared device and is not intended to be installed, formatted, rebuilt, or provisioned by end users.

The USB contains the platform apps, official-device metadata, and Boot OS components required for supported workflows. Public documentation describes the device model and trust boundaries; it does not provide manufacturing, provisioning, or installation instructions.

## Overview

The official USB uses a controlled multi-partition layout:

  <Card title="Boot boundary">
    EFI boot material for Enigm Link Boot OS.
  </Card>
  <Card title="Visible app boundary">
    User-visible Enigm Link volume with desktop apps and official device metadata.
  </Card>
  <Card title="Boot OS boundary">
    Root filesystem material for the bootable Enigm Link environment.
  </Card>
</CardGroup>

The visible app boundary is critical. It contains the desktop apps, official device metadata, and user-facing platform folders. Boot OS update workflows must preserve this boundary on the official USB.

## Visible Content Model

The visible USB content should remain simple and predictable:

- `macOS`: shows only the macOS app.
- `Windows`: shows only the Windows executable.
- `Linux`: shows only the Linux executable.

Auxiliary files, staging data, update backups, autorun files, and internal metadata should remain hidden where supported by the platform. Users should not need to inspect or modify internal files.

## Official Device Metadata

Enigm Link uses official-device metadata to validate the USB and support lifecycle operations.

Official metadata supports:

- Device validation.
- Update eligibility.
- Controlled update workflows.
- Product lifecycle state.
- Enigm Command registration where applicable.

Official metadata must be preserved during app updates and Boot OS updates. Update workflows must not treat the visible app boundary as disposable storage.

## Closed Product Model

Enigm Link is delivered as a closed official USB product.

Users and customers should not:

- Repartition the USB.
- Format the visible volume.
- Replace Boot OS partitions manually.
- Rebuild the device from public files.
- Modify official metadata.
- Treat internal files as user-managed configuration.

Supported lifecycle actions are performed through Enigm Link itself and governed through Enigm Command. This keeps official-device identity, update eligibility, and protected-state behavior tied to Enigm's product lifecycle controls.

## Security Boundaries

The visible app boundary and Boot OS boundaries have different responsibilities.

- The visible app boundary stores platform apps and official-device metadata.
- Boot OS boundaries support boot and isolated runtime behavior.
- Boot OS updates must preserve visible app content and official-device identity.
- App updates must preserve official metadata and avoid leaving incomplete visible folders.
