Enigm Command is the control surface for supported product lifecycle operations across the Enigm ecosystem. Product lifecycle state is commercial and administrative state; it is not protected communication access.

## Overview

Product lifecycle workflows include:

- Enigm account creation.
- Enigm account lifecycle and deletion.
- Critical-operation PIN authorization for high-impact lifecycle actions.
- Supported payment workflows through cryptocurrency, credit card, and Code Coin payment methods.
- Product activation and entitlement review.
- Enigm Link purchase, official USB registration, update eligibility, lifecycle review, and support workflows.
- Enigm Server purchase, creation, region selection, join request approval, membership, content lifecycle, and deletion.
- Enigm eSIM purchase, activation lifecycle, account association, unlinking, deletion, and retirement.
- Enigm Key device visibility, loss handling, revocation, and replacement.
- Enigm OS managed-device lifecycle when the user activates managed-device mode.
- Enyra Product Assistant guidance for account, device, product, configuration, and navigation workflows.
- Essential or advanced training with final-exam workflows.

## Enigm Link Management

Enigm Command supports Enigm Link as a governed sub-product lifecycle.

Enigm Link workflows include:

- Purchasing or activating Enigm Link.
- Reviewing product entitlement.
- Registering official USB devices where applicable.
- Reviewing official-device lifecycle state.
- Supporting controlled update eligibility.
- Supporting app and Boot OS lifecycle visibility.
- Supporting product troubleshooting and support workflows.

Enigm Link lifecycle state is product and device lifecycle state. It does not provide access to message plaintext, secure call content, browser session content, private key material, or protected Enigm App communications.

See [Enigm Link](/link/overview) for the Enigm Link product documentation.

Enigm Command supports Enigm Server purchase, creation, and administration.

Server management workflows include:

- Purchasing or activating Enigm Server.
- Creating dedicated private messaging environments.
- Assigning or reviewing server ownership inside the authorized Enigm Command boundary.
- Selecting a geographic deployment region.
- Displaying the server ID used by users to request access.
- Reviewing and accepting join requests.
- Removing approved users from the server environment.
- Managing server membership.
- Applying server-scoped encrypted content lifecycle controls.
- Deleting server-scoped encrypted objects according to policy.
- Deleting all encrypted content within the dedicated server environment.
- Deleting the entire server environment.

Enigm Server management must preserve the separation between administrative control and protected communication confidentiality. Administrators can manage the lifecycle and availability of server-scoped encrypted content, but administrative controls do not grant access to message plaintext, attachment plaintext, user communications, or private key material.

## Enigm eSIM Management

Enigm Command supports Enigm eSIM purchase and lifecycle workflows.

Enigm eSIM management workflows include:

- Purchasing Enigm eSIM.
- Activating Enigm eSIM.
- Reviewing Enigm eSIM status.
- Managing activation lifecycle.
- Reviewing Enigm account association.
- Supporting unlinking, deletion, replacement, or retirement workflows.
- Applying managed connectivity policy where applicable.

Enigm eSIM state is connectivity state. It does not provide access to message plaintext, call content, media content, attachments, user conversations, or private key material.

## Enigm Key Management

Enigm Command supports Enigm Key administration as an associated device when the user or deployment enables emergency-device workflows.

Enigm Key workflows include:

- Associated Enigm Key visibility.
- Device lifecycle visibility.
- Device loss handling.
- Device revocation.
- Device replacement.
- Emergency event visibility where authorized.

Enigm Key initial linking and emergency contact configuration are controlled from Enigm App. Enigm Command is not the initial linking surface or emergency contact configuration surface.

## Commercial Boundaries

Payment state, entitlement state, product activation state, and subscription state are commercial lifecycle states. They do not provide access to protected communications, private key material, device-held cryptographic material, or message plaintext.

## Training Lifecycle

Enigm Command can provide essential and advanced training workflows for Enigm users.

Training workflows can include product education, security guidance, configuration guidance, and a final exam. Training status is account education state; it does not provide access to protected communications, private key material, or administrative decryption capability.

See [Payment Privacy](/command/payments) for the payment model.
