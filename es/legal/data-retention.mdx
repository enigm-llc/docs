Enigm está diseñado para recoger y conservar solo la información necesaria para operar servicios, mantener seguridad y cumplir obligaciones legales.

## Principios de retención

- Retención mínima.
- Limitacion de propósito.
- Reduccion de riesgo.
- Seguridad operaciónal.
- Control del usuario.

## Datos de cuenta

Los datos de cuenta se conservan mientras dura el servicio y hasta que el usuario elimina la cuenta. La eliminación solicitada por el usuario se ejecuta de forma inmediata.

## Datos de dispositivo

Los datos de dispositivos asociados se conservan mientras el usuario mantiene el dispositivo vinculado o gestionado.

## Sesiones

Las sesiones de Enigm App tienen una duracion de 6 horas.

## Mensajes y adjuntos

Mensajes, archivos, imágenes, videos y multimedia tienen vida máxima de 30 dias. El usuario o la conversacion puede definir un período menor, y el usuario puede borrar contenido manualmente de forma inmediata.

## Enigm Server

El entorno Enigm Server se conserva mientras el usuario lo mantiene activo. El contenido dentro del servidor sigue el mismo modelo de mensajes: máximo 30 dias o menor segun política.

## Enigm Key

Los eventos de emergencia se tratan como mensajes/eventos protegidos. La ubicación se comparte hasta que el usuario cancela el envío.

## Enigm eSIM

El estado de Enigm eSIM se conserva hasta que el usuario elimina o desvincula la eSIM.

## Datos de seguridad y operación

Los eventos de seguridad y datos operativos se conservan hasta un máximo de 30 dias cuando son necesarios para seguridad, fiabilidad o fraude.

Los casos de seguridad se conservan 30 dias desde el cierre del caso.

## Active Defense

Active Defense no retiene datos de análisis. Puede conservar metadatos de resultado si el usuario decide mantenerla.
