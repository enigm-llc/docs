Enigm utiliza un modelo de defensa en profundidad para proteger privacidad, comunicaciones, dispositivos, administración, software delivery y operaciones.

## Resumen

La seguridad existe para sostener privacidad: confidencialidad de contenido, minimización de datos, identidad minimizada, reducción de metadatos y control del usuario.

## Objetivos de seguridad

- Proteger mensajes, llamadas, medios y adjuntos.
- Separar administración de contenido.
- Mantener claves privadas ligadas a dispositivos.
- Reducir exposición de identidad y metadatos.
- Proteger releases y actualizaciones.
- Aportar visibilidad de seguridad sin inspeccionar contenido.

## Límites de seguridad de producto

### Enigm

Producto de mensajería privada con cifrado de extremo a extremo, llamadas seguras, Active Defense, multi-dispositivo y privacidad de red.

### Enigm Command

Panel web de ciclo de vida. Gestióna cuentas, dispositivos, productos, sesiones y operaciones administrativas. No lee contenido protegido.

### Enigm Server

Entorno dedicado. El administrador gestiona membresía y disponibilidad de contenido cifrado, no texto claro.

### Enigm OS

Capa opcional de endpoint seguro con Trust Security Center, hardening, política de red y OTA Architecture.

### Enigm eSIM y Enigm Key

Enigm eSIM aporta conectividad de datos. Enigm Key aporta emergencia física y ubicación vinculada al evento.

## Separación de confianza

Account Trust, Device Trust, Trust Security Center, Remote Attestation, OTA Eligibility y Administrative Authorization son conceptos separados.

## Gobernanza

Enigm mantiene certificación ISO/IEC 27001:2022, revisiónes recurrentes, evaluaciones de vulnerabilidad, pruebas adversariales y mejora continua.
