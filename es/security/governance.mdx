Enigm mantiene un modelo formal de gobernanza de seguridad orientado a seguridad, privacidad, resiliencia operativa y mejora continua.

## ISO 27001

Enigm mantiene certificación ISO/IEC 27001:2022 para el alcance certificado documentado. El alcance cubre el sistema de gestión de seguridad de la información que soporta actividades de desarrollo de la aplicación de mensajería cifrada, infraestructura de desarrollo relaciónada, políticas internas y procesos incluidos en la Declaración de Aplicabilidad.

[Download public ISO 27001 certificate](https://raw.githubusercontent.com/enigm-llc/docs/main/assets/compliance/enigm-iso-27001-certificate.pdf)

## Risk Management

Los riesgos se identifican, evaluan, priorizan y tratan mediante procesos estructurados.

## Independent Assessments

Enigm mantiene evidencia privada bajo NDA, incluyendo auditoría criptográfica, pentest, auditoría móvil, auditoría de infraestructura y revisiónes de seguridad.

## Ongoing Security Validation

Enigm realiza validacion continua y periodica:

- Vulnerability assessments.
- Infrastructure exposure reviews.
- Security posture validation.
- Configuration reviews.
- Attack surface monitoring.
- Security control validation.
- Adversarial security testing.
- Simulated attack exercises.
- Security review cycles.

## Compliance

Enigm alínea su programa con marcos reconocidos de seguridad, revisiónes recurrentes, evaluaciónes independientes y mejora continua.

## Cryptographic Assurance

Enigm incorpora algoritmos criptográficos post-cuánticos estandarizados por NIST como parte de su arquitectura criptográfica.

No se afirma certificación, aprobacion o auditoría de Enigm por NIST.

## Transparency

Enigm pública información de seguridad cuando es apropiado, equilibrando transparencia, precision y seguridad operativa.

Consulta [Transparency Report](/es/security/transparency-report) y [Platform Limitations](/es/legal/limitations).
