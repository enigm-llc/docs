La privacidad es una propiedad fundacional de Enigm, no una función opcional.

## Privacidad desde el diseño

Las decisiones de arquitectura se evaluan desde el inicio con foco en minimización de datos, identidad minimizada, reducción de metadatos y control del usuario.

## Minimización de datos

Enigm recopila y conserva solo la información necesaria para operar servicios, mantener seguridad, preservar integridad de plataforma y cumplir obligaciones legales.

## Minimización de identidad

El registro estándar no requiere email, teléfono ni documento de identidad. Enigm reduce dependencia de identificadores públicos donde es posible.

## Reducción de metadatos

Enigm combina:

- Identificadores privacy-preserving.
- Proxy Network.
- VPN Service.
- Actividad de red aleatoria.
- Retención limitada.
- Separacion de tráfico.
- Controles de acceso.
- Cifrado parcial de metadatos donde el dominio lo soporta.

## Confidencialidad del contenido

Los sistemas administrativos no proporcionan acceso a mensajes, llamadas, adjuntos, multimedia, conversaciones ni claves privadas en texto claro.

## Control del usuario

El usuario controla cuenta, dispositivos, mensajes, expiracion, borrado, Enigm Key, Enigm eSIM y gestión de productos desde Enigm App y Enigm Command segun corresponda.

## La seguridad como habilitador de privacidad

Controles como cifrado de extremo a extremo, Device Trust, Remote Attestation, Hardware-Backed Signing, OTA Architecture y Trust Security Center existen para reforzar privacidad.

Consulta [Retención de datos](/es/legal/data-retention) y [Limitaciones de plataforma](/es/legal/limitations).
