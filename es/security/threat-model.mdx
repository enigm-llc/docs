El threat model de Enigm cubre Enigm, Enigm Command, Enigm Server, Enigm OS, Trust Security Center, OTA Architecture, Remote Attestation, Enigm Intelligence, Enyra, VPN Service, Proxy Network, Tor Gateway, Hardware-Backed Signing y Controlled Device Management.

## Threat Domains

Enigm evalua amenazas en dominios separados:

- Cuenta e identidad.
- Dispositivo y endpoint.
- Mensajería y llamadas.
- Administración.
- Enigm Server.
- Enigm Key.
- Enigm eSIM.
- Red y metadatos.
- OTA y signing.
- Inteligencia y respuesta defensiva.

## Trust Boundaries

El modelo separa Account Trust, Device Trust, administración, Remote Attestation, OTA Eligibility y contenido protegido.

## Expected Controls

Los controles incluyen cifrado de extremo a extremo, claves protegidas, Privacy-Preserving Device Handles, minimización de metadatos, revocacion, logging limitado, seguridad de releases, governance y revisión de riesgos.

## Administrative Abuse

Enigm Command y Enigm Server son superficies administrativas. Deben permitir ciclo de vida, no acceso a texto claro ni claves privadas.

## Network Observation

VPN Service, Proxy Network y actividad de red aleatoria reducen exposición y correlación simple, pero no garantizan anonimato absoluto.

Consulta [Platform Limitations](/es/legal/limitations).
