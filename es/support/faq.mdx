Una colección de preguntas habituales sobre el ecosistema Enigm.

<AccordionGroup>
  <Accordion title="¿Qué es Enigm?">
    Enigm es el producto de mensajería privada dentro del ecosistema Enigm. Proporciona mensajería segura, llamadas seguras, onboarding de cuenta, asociación de dispositivos, Active Defense, flujos multi-dispositivo, uso de VPN Service y separación de tráfico mediante Proxy Network.
  </Accordion>

  <Accordion title="¿Qué es Enigm Command?">
    Enigm Command es el panel web para ciclo de vida de cuenta, visibilidad de dispositivos, control de sesiones, gestión de Enigm Server, gestión de Enigm eSIM, visibilidad de Enigm Key, ciclo de vida de producto, flujos de pago y operaciones de dispositivos gestionados.

    Enigm Command no es un cliente de mensajería y no proporciona acceso en texto claro a mensajes, adjuntos, llamadas o claves privadas.
  </Accordion>

  <Accordion title="¿Qué es Enigm Server?">
    Enigm Server es un entorno dedicado de mensajería privada para usuarios aprobados de Enigm. Soporta membresía controlada, solicitudes de unión mediante ID de servidor, aprobación por administrador, selección de región y control del ciclo de vida de contenido cifrado dentro del servidor.

    Los administradores de servidor pueden gestionar membresía y disponibilidad de contenido cifrado. No pueden leer texto claro de mensajes, adjuntos, comunicaciones de usuarios o material de claves privadas.
  </Accordion>

  <Accordion title="¿Qué es Enigm OS?">
    Enigm OS es un sistema operativo seguro diseñado para aportar Device Trust adicional, endurecimiento de plataforma, visibilidad mediante Trust Security Center, controles de política de red, gestión controlada de dispositivos y gobernanza de actualizaciones seguras.

    Enigm OS refuerza la postura del dispositivo, pero no sustituye al cifrado de extremo a extremo, las decisiones de confianza del usuario, la arquitectura de mensajería segura o la concienciación de seguridad.
  </Accordion>

  <Accordion title="¿Qué es Enigm Key?">
    Enigm Key es un dispositivo físico de emergencia. Permite activar un flujo SOS que notifica a contactos seleccionados y comparte información de ubicación vinculada al evento hasta que el usuario cancela el flujo de envío de emergencia.

    Enigm Key permanece dormido durante el funcionamiento normal sin emergencia para apoyar la privacidad del usuario.
  </Accordion>

  <Accordion title="¿Qué es Enigm eSIM?">
    Enigm eSIM es un producto de conectividad privada centrado en datos móviles de solo datos. Se compra y gestiona mediante Enigm Command y se vincula a la cuenta Enigm del usuario.

    Enigm eSIM está separado de la mensajería segura, el cifrado de extremo a extremo, VPN Service, Proxy Network y Device Trust.
  </Accordion>

  <Accordion title="¿Es Enigm el operador móvil de Enigm eSIM?">
    No. Enigm proporciona Enigm eSIM como servicio de facilitación comercial y gestión de ciclo de vida. Enigm no es operador móvil, operador móvil virtual, proveedor de telecomunicaciones, operador de red radioeléctrica ni emisor directo de la conectividad subyacente.

    El servicio de datos móviles subyacente es operado por un proveedor independiente de infraestructura de telecomunicaciones.
  </Accordion>

  <Accordion title="¿Puede Enigm proporcionar logs de red del operador para Enigm eSIM?">
    Enigm no opera la red del operador y no recibe registros de tráfico de capa carrier como parte del funcionamiento normal de Enigm eSIM.

    Enigm no puede proporcionar registros de tráfico de red del operador, logs de asignación IP del operador, logs de conexión del operador o registros de telecomunicaciones mantenidos solo por un proveedor independiente de infraestructura de telecomunicaciones.
  </Accordion>

  <Accordion title="¿Enigm requiere email, teléfono o documento de identidad para el registro estándar?">
    El registro estándar de cuenta Enigm está diseñado para minimizar la dependencia de identificadores públicos. Enigm no requiere email, número de teléfono o documento de identidad para el registro normal de cuenta.
  </Accordion>

  <Accordion title="¿Puede Enigm leer mensajes o adjuntos?">
    Los sistemas administrativos de Enigm no están destinados a proporcionar acceso en texto claro a mensajes, adjuntos, contenido de llamadas seguras, medios o conversaciones de usuarios.

    El almacenamiento de mensajes en servidor, cuando se requiere para entrega, almacena objetos de contenido cifrado.
  </Accordion>

  <Accordion title="¿Están protegidos los metadatos e identificadores operativos?">
    Los metadatos e identificadores operativos se minimizan, limitan por propósito, protegen mediante controles de acceso y se cifran parcialmente donde el producto o dominio de almacenamiento correspondiente lo soporta. Enigm utiliza identificadores privacy-preserving donde se requiere correlación de dispositivo o ciclo de vida.

    Algunos identificadores operativos siguen siendo necesarios para enrutar servicios, autenticar solicitudes, aplicar controles de ciclo de vida, soportar monitorización de seguridad y mantener integridad de plataforma.
  </Accordion>

  <Accordion title="¿Cómo protege Enigm los mensajes?">
    La mensajería de Enigm utiliza cifrado de extremo a extremo. El acceso a mensajes depende de asociación de dispositivo de confianza y material criptográfico protegido. Los controles administrativos no otorgan acceso al texto claro de mensajes.
  </Accordion>

  <Accordion title="¿Enigm usa criptografía post-cuántica?">
    Enigm incorpora algoritmos criptográficos post-cuánticos estandarizados por NIST como parte de su arquitectura criptográfica.

    Esta afirmación no significa que Enigm esté certificado, aprobado o auditado por NIST.
  </Accordion>

  <Accordion title="¿Qué es Active Defense?">
    Active Defense es una capacidad de seguridad de Enigm disponible en producción para un grupo seleccionado de usuarios. Analiza comportamiento de red relevante para seguridad durante un período de evaluación acotado para ayudar a identificar patrones sospechosos de malware móvil o riesgo de spyware.

    Active Defense no inspecciona texto claro de mensajes, contenido de llamadas, adjuntos, medios, documentos, conversaciones de usuarios, claves privadas o frases de recuperación.
  </Accordion>

  <Accordion title="¿Pueden los administradores saltarse el cifrado?">
    No. La autoridad administrativa puede gestionar ciclo de vida, membresía, dispositivos, sesiones, estado de producto y disponibilidad de contenido cifrado cuando está autorizada. No proporciona autoridad criptográfica ni acceso en texto claro.
  </Accordion>

  <Accordion title="¿Está Enigm certificada ISO 27001?">
    Sí. Enigm mantiene certificación ISO/IEC 27001:2022 para el alcance certificado documentado. El alcance certificado cubre el sistema de gestión de seguridad de la información que soporta actividades de desarrollo de la aplicación de mensajería cifrada, incluyendo gobernanza de desarrollo relacionada, infraestructura de desarrollo de soporte, políticas internas de seguridad y procesos de la compañía incluidos en la Declaración de Aplicabilidad.
  </Accordion>
</AccordionGroup>
