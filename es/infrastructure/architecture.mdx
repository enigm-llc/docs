Esta sección describe la arquitectura del sistema de soporte utilizando únicamente términos genéricos. Enmarca la infraestructura como un plano de control que respalda la privacidad para los productos Enigm en lugar de un mapa público de sistemas operativos.

## Componentes

- Edge runtime
- Capa de identidad segura
- Almacén de metadatos
- Almacenamiento de objetos privado
- Pipeline de inteligencia de amenazas
- Sistema de despliegue controlado
- Auditoría y seguimiento de flujos de trabajo.

## Objetivos de seguridad

Los sistemas de soporte deben imponer privilegios mínimos, minimización de datos, cambios controlados, auditabilidad y verificación de integridad.

La topología de implementación no se divulga intencionalmente en la documentación pública. La documentación de la infraestructura pública se centra en los objetivos de seguridad, los límites de la privacidad, la minimización de datos, la auditabilidad, la resiliencia y la garantía empresarial.

## Seguridad del transporte web público

Las superficies web públicas utilizan una base estricta de seguridad en el transporte.

A nivel de arquitectura pública, esto incluye:

- Certificados TLS para nombres de host proxy públicos.
- Gestión automatizada del ciclo de vida de los certificados para nombres web públicos.
- TLS 1.3 protección de transporte preferente.
- Compatibilidad de TLS 1.2 restringida a conjuntos de cifrado seguros aprobados.
- Seguridad de transporte estricta HTTP.
- HSTS edad máxima de 6 meses.
- Cobertura de subdominio HSTS.
- Hola de cliente cifrado para rutas de acceso web públicas compatibles.

La seguridad del transporte web público protege las rutas de acceso web y los metadatos de transporte admitidos. No reemplaza el cifrado de extremo a extremo Enigm App, Device Trust, el material de claves protegido ni los flujos de trabajo de verificación.

La documentación pública no revela nombres de autoridades de certificación, nombres de relaciones de borde público, nombres de host internos, dominios no públicos, topología de enrutamiento privado, elementos internos de automatización de certificados ni detalles de implementación operativa.
